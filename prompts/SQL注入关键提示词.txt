你这次的目标是读取得到的所有文件，并找到该系统中的所有 SQL 注入漏洞。
该系统的厂商已经在本地搭建好测试系统，并映射为：
http://cvehunter.test/{提供的项目文件夹}
因此，后续需要输出的漏洞信息都要包含与该 URL 相关的路径信息。

输出要求与规则（请务必严格遵守）：
0、每当你发现一个 SQL 注入漏洞，就按下列“统一输出模板”完整输出一段。
    - 不需要解释原因，不要输出推理过程；
    - 只输出与要求最接近、简明易读的结果；
    - 禁止使用 Markdown 代码块（```）与反引号（`...`）；所有行均为纯文本。
1、统一输出模板（逐行输出，保留编号与冒号，不要缺省任何一行）：
—— 第<N>个相关漏洞 ——
【15】<中文权限：从「无需登录或授权即可利用此漏洞」「需要基本用户权限才能利用此漏洞」「需要管理员权限才能利用此漏洞」中选一项>

【Type】:<GET|POST>
【URL】<完整URL（形如 http://cvehunter.test/{提供的项目文件夹}/...）>
【9】漏洞参数：<以分号分隔的参数名；若无参数请写 “No parameters”>
【12】sqlmap（快速）：<根据【Type】生成命令；单行输出，无注释>
==============================

SQLMap 命令生成规范（必须遵守）：
- 总则：
  - 命令必须为单行纯 ASCII 文本；不要在 URL 外层加反引号；不要在命令末尾附加中文说明或杂项字符。
  - 【9】中的参数名用于 -p，分隔符统一使用英文逗号，例如：-p "id,name,email"。
- 对于 GET：
  - 在 URL 中附带所有要测试的查询参数并设置占位值 1，例如："http://.../path.php?id=1&name=1"。
  - 示例（快速）：python sqlmap.py -u "<URL带查询参数>" -p "<以逗号分隔的参数>" --batch --level 5 --risk 3 --dbs --time-sec=3
- 对于 POST：
  - 必须显式指定 --method=POST 与 --data，--data 中包含表单字段并设置占位值，例如："id=1&name=1"。
  - 建议同时添加头：--headers "Content-Type: application/x-www-form-urlencoded"。
  - 示例（快速）：python sqlmap.py -u "<URL>" --method=POST --data "<k1=1&k2=1&...>" -p "<以逗号分隔的参数>" --batch --level 5 --risk 3 --dbs --time-sec=3 --headers "Content-Type: application/x-www-form-urlencoded"
- 登录/会话：
  - 如需登录会话，优先在快速命令中附加 --cookie "PHPSESSID=<占位>"；不再使用 -r 方式。
  - 示例：python sqlmap.py -u "<URL或登录后页面>" -p "<以逗号分隔的参数>" --batch --level 5 --risk 3 --dbs --time-sec=3 --cookie "PHPSESSID=<占位>"
- 其它：
  - 如存在 CSRF/隐藏字段，快速方式通常不拼接；如需会话信息，可附加 --cookie "PHPSESSID=<占位>"。
  - 不允许在 -p 中出现非参数内容（如中文说明、标点）。

注意事项：
- 每个漏洞独立成段输出，各段之间空一行；
- 除第一行与最后一行分割线外，所有行必须以指定编号开头；
- 路径与参数名应从代码中精确提取（例如 PHP 中的 $_GET/$_POST、路由控制器、SQL 构造位置）。
- 同页面、同参数的漏洞认定为同一个漏洞，不允许重复输出；若存在多处相同类型注入，请分别输出每一处。
